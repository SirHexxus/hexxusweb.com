The Premise

Why a style guide conditions an AI

  • → What makes a style guide good for humans - explicit rules, clear rationale, worked examples of right and wrong - is exactly what makes it good context for a coding model. These were written for human developers first. The AI benefit fell out of the rigor.
  • → They're opinionated on purpose: snake_case for JavaScript functions, BEM for CSS, data-attributes as the HTML-JS bridge, pathlib over os.path, explicit error checks over set -e. Hedged guides produce hedged output. These don't hedge.
  • → They're in daily production use conditioning Claude Code on real client and infrastructure projects - including the site you're reading right now.
